# Implementation Plan: P2P Data Delivery

**Branch**: `009-p2p-data-delivery` | **Date**: 2026-03-26 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/009-p2p-data-delivery/spec.md`

## Summary

Spec 009 defines the data plane for Neuron agent-to-agent service delivery. It consumes Spec 008's control-plane hooks (`delivery` descriptor, `connectionSetup` message, encryption requirement FR-P34) and specifies how delivery channels are established, maintained, and torn down via libp2p. The implementation covers: abstract DeliveryAdapter interface (5 operations), ECIES multiaddr encryption (secp256k1 ECDH + HKDF + AES-256-GCM), connection lifecycle state machine (6 states), NAT traversal (AutoNAT + Relay + DCUtR), length-prefixed stream framing, and multi-transport support (QUIC + WebRTC).

## Architecture Decisions

**DD-D01**: ECDH is self-contained in `delivery/ecies.go` using `go-ethereum/crypto` directly, not added to keylib (Spec 002). Keeps ECIES co-located with its only consumer. Avoids spec amendment.

**DD-D02**: libp2p host is created per-agent (not per-channel) and shared across delivery channels. Configured with QUIC + WebRTC + AutoNAT + Relay + DCUtR.

**DD-D03**: ConnectionState machine is local runtime state, not protocol-level. Different SDKs under different network conditions compute different states. SC-D03 tests determinism given identical transport event sequences.

**DD-D04**: Length-prefix framing (FR-D22) wraps application data on libp2p streams. Implemented as `FrameReader`/`FrameWriter` wrapping `io.Reader`/`io.Writer`.

**DD-D05**: Backoff parameters (5s/2x/10min/1hr) are defaults in a `BackoffConfig` struct. Configurable for testing.

## 2026-05-08 Amendment — Plan Addendum

Adds plan coverage for FR-D11a, FR-D-stream-direction, FR-D-multi-protocol, FR-D-wildcard-handler, FR-D-pubsub-primitives, FR-D-degraded.

### New Architectural Decisions

**DD-D05** (multiaddr filtering): Filter rules in FR-D11a apply at ENCRYPTION time (publisher side, before ECIES wraps the array) AND at DECRYPTION time (receiver side, before passing to `connect`). Implementation surface: a single `filterMultiaddrs(addrs []multiaddr.Multiaddr) []multiaddr.Multiaddr` helper in `internal/delivery/multiaddr_filter.go` consumed by both `BuildConnectionSetup` and `ConnectFromSetup`. Interface-name detection is OS-specific — wrap behind `runtime.GOOS` switch. The Go reference implementation uses `net.Interfaces()` to map address → interface name.

**DD-D06** (stream direction): The stream-direction enforcement lives in the libp2p adapter. New primitive: `RegisterStream(protocolID string, direction StreamDirection, handler StreamHandler)`. The adapter rejects `OpenStream` calls from the wrong-direction party with `StreamDirectionViolation`. The legacy single-string `Protocol` path defaults to `direction = "seller-initiates"` for back-compat.

**DD-D07** (multi-protocol + wildcard handler): libp2p's `Host.SetStreamHandler` accepts a single protocol ID; for wildcards, register a `StreamHandlerMatcher` (`Host.SetStreamHandlerMatch` in go-libp2p) that pattern-matches on the protocol ID prefix. Wildcard registration: `RegisterStreamPattern(prefix string, handler ParameterizedStreamHandler)` where the handler receives the matched parameter as a string.

**DD-D08** (pubsub primitives): Re-export libp2p's `pubsub.PubSub` and `gossipsub.GossipSub` from `internal/delivery/pubsub.go` without prescribing topology. The package exposes `NewPubSub(host)` and `NewGossipSub(host, opts...)` constructors that pass through libp2p configuration unchanged. NO mesh-parameter defaults are encoded — DApps configure their own.

**DD-D09** (degraded mode): Mirror DD-P07. Audit data-plane code paths in `internal/delivery/libp2p_adapter.go` for any control-plane (topic backend) dependency; remove or downgrade. The data-plane connection lifecycle is governed solely by libp2p's transport state, NOT by topic-backend availability.
