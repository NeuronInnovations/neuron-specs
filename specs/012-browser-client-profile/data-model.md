# Data Model: Browser Client Profile (012)

**Source**: spec.md Key Entities + FR-B01 through FR-B36; research.md R0.1–R0.12.

All entities are in-memory within a single page lifecycle. Nothing here maps to persistent storage (enforced by FR-B03 / FR-B26 / FR-B27).

---

## BootstrapJSON

Static same-origin descriptor generated by the seller at startup and served next to the browser bundle. Browser refuses any bootstrap whose `version` is not `1` or that contains unknown top-level keys.

| Field | Type | Required | Description | Source FR |
|-------|------|----------|-------------|----------|
| `version` | number (`1`) | ✓ | Schema version. v1 demo always `1`. | R0.4 |
| `sellerEVMAddress` | string (EIP-55 checksummed hex) | ✓ | Seller's expected EVM address; pinned for sender verification. | FR-B14, FR-B23 |
| `sellerPeerID` | string (base58btc, `12D3KooW…`) | ✓ | Seller's expected libp2p PeerID; enforced on Noise handshake. | FR-B06, FR-B23 |
| `sellerWSSMultiaddr` | string (`/ip4/…/tcp/PORT/ws/p2p/PEERID` or `/dns4/…/tcp/443/wss/p2p/PEERID`) | ✓ | Dial address. `/ws` allowed only when page origin is `http://localhost[:PORT]`; `/wss` required otherwise. | FR-B05, FR-B09 |
| `controlStreamProtocolID` | string | ✓ | libp2p stream protocol ID for in-stream TopicAdapter (Phase 1: `"/neuron/browser-profile/control/1.0.0"`). | FR-B11 |
| `dataStreamProtocolID` | string | ✓ | libp2p stream protocol ID for file delivery (Phase 1: `"/neuron/browser-profile/data/1.0.0"`). | FR-B11 |

**Validation rules**:
- `version` strict equality with `1`; any other value → configuration error `NEURON-BROWSER-002`.
- Unknown top-level key → `NEURON-BROWSER-003`.
- Any required field missing → `NEURON-BROWSER-004`.
- `sellerEVMAddress` fails EIP-55 checksum validation → `NEURON-BROWSER-005`.
- `sellerPeerID` fails base58btc decode or multihash check → `NEURON-BROWSER-006`.
- `sellerWSSMultiaddr` scheme is not `/ws` or `/wss` → `NEURON-BROWSER-007`.
- Scheme `/ws` but page origin is not `http://localhost[:PORT]` → `NEURON-BROWSER-008` (prevents accidental non-loopback ws:// use).

## BrowserSession

In-memory context for a single page lifecycle. One session at a time; FR-B28 prevents concurrent "Buy".

| Field | Type | Mutable | Description | Source FR |
|-------|------|---------|-------------|----------|
| `privateKey` | Uint8Array (32 bytes) | no | secp256k1 private key. Lives in a closure. Never returned from exported APIs, never logged. | FR-B01, FR-B04 |
| `publicKey` | Uint8Array (33 bytes, compressed) | no | Derived. | FR-B02 |
| `evmAddress` | string (EIP-55) | no | Derived. | FR-B02 |
| `peerID` | string (base58btc) | no | Derived. | FR-B02 |
| `didKey` | string (`did:key:z…`) | no | Derived. | FR-B02 |
| `host` | `Libp2p` handle | no | Active js-libp2p instance. | FR-B05 |
| `state` | `BuyerState` | yes | See below. | FR-B15 |
| `ledger` | `LedgerEntry[]` | yes (append-only) | Signature-chain record. | FR-B29 |
| `abortController` | `AbortController` | no | Fired on tab close / new "Buy" / error. | FR-B26, R0.R7 |

**Lifecycle rules**:
- Created on the first "Buy" click after page load (not at page load itself, so the keypair isn't generated unless needed).
- Destroyed on: tab `beforeunload`, any transition to `aborted`, or any subsequent "Buy" (FR-B36 — new session, new keypair).
- `privateKey` MUST NOT leave the scope of `session.ts`. All signing and ECDH operations are methods that take data in and return only public results.

## BuyerState (state machine)

6 active states + 1 terminal-abort state. Matches spec.md Key Entities.

```
  ┌──────┐  Buy clicked  ┌──────────────┐
  │ idle │ ───────────▶ │ request-sent │
  └──────┘               └──────┬───────┘
                                │ paymentDetails received + verified
                                ▼
                         ┌───────────────┐
                         │ quote-received│
                         └──────┬────────┘
                                │ connectionSetup received + verified + decrypted
                                ▼
                   ┌────────────────────────────┐
                   │ connection-setup-received  │
                   └────────────┬───────────────┘
                                │ invoiceAck sent
                                ▼
                        ┌────────────────┐
                        │invoice-ack-sent│
                        └──────┬─────────┘
                               │ file fully received + SHA-256 verified
                               ▼
                         ┌──────────┐
                         │ complete │
                         └──────────┘

 any state ──── abort trigger (see below) ────▶  ┌──────────────────────────┐
                                                 │ aborted { reason, code } │
                                                 └──────────────────────────┘
```

**Abort triggers** (each wired to an error code in the `NEURON-BROWSER-NNN` taxonomy):

| Trigger | Code range | FR |
|---|---|---|
| Configuration fault (bootstrap / unsupported browser) | 001–019 | FR-B23, FR-B24 |
| Transport fault (TLS, dial, port) | 020–039 | FR-B05, FR-B07 |
| Handshake fault (Noise, PeerID mismatch) | 040–059 | FR-B06, FR-B07 |
| Signature verification fault | 060–079 | FR-B13, FR-B14 |
| SHA-256 mismatch | 080–099 | FR-B17, FR-B22 |
| Size cap / framing fault | 100–119 | FR-B19, FR-B20, FR-B21 |
| Timeout (read-idle / deadline) | 120–139 | FR-B34 |
| State-machine protocol fault (out-of-order message) | 140–159 | FR-B15 |

**Invariants**:
- Each transition is a pure function of `(currentState, receivedEnvelope or userAction)`.
- Any verification failure in a transition MUST abort, not downgrade or retry.
- `aborted` is terminal for the current session; the user must click "Buy" again to start a new one (which triggers FR-B36 rotation).

## LedgerEntry

Append-only per-session record, rendered into DOM by `ui/ledger.ts`.

| Field | Type | Description | Source FR |
|-------|------|-------------|----------|
| `direction` | `"inbound" \| "outbound"` | Whether the browser sent or received this envelope. | FR-B29 |
| `messageType` | string | `"serviceRequest"`, `"paymentDetails"`, `"connectionSetup"`, `"invoiceAck"` (Phase 1 set). | FR-B15 |
| `senderAddress` | string (EIP-55) | Envelope's recovered sender. | FR-B29 |
| `timestamp` | number (ms epoch) | Envelope's declared timestamp. | FR-B29 |

| `failureCode` | string \| undefined | `NEURON-BROWSER-NNN` if `signatureStatus === "failed"`. | FR-B29, FR-B35 |

## InStreamTopicAdapter (wire-level)

A Spec 004 `TopicAdapter` implementation whose transport is a single libp2p stream opened on the control stream protocol. Symmetric: browser and Node.js seller each implement one.

### Operations

| Operation | Signature | Description |
|-----------|-----------|-------------|
| `open(host, remotePeerID)` | `() → Promise<StreamHandle>` | Dial the control stream on the remote peer; fail if the stream protocol ID doesn't match. |
| `publish(envelope)` | `(TopicMessage) → Promise<void>` | Sign (if not already signed) → canonical-JSON encode → length-prefix → write to stream. |
| `subscribe(handler)` | `(fn: TopicMessage → void) → unsub` | Read length-prefixed frames from the stream; parse + verify signature + call handler. Silently discards keep-alive (zero-length) frames. |
| `close(reason)` | `(string) → Promise<void>` | Close the stream, emit a ledger entry with direction `"outbound"` and type `"control-close"`. |

### Wire contract (shared between browser + Node)

- Every message is a canonical-JSON-encoded Spec 004 `TopicMessage` envelope, length-prefixed with a 4-byte big-endian uint32.
- One control stream per session (not one stream per message).
- Zero-length frame = keep-alive, silently consumed.
- Abnormal stream close (before `complete`) MUST abort with `NEURON-BROWSER-021` (transport: stream closed unexpectedly).

### State

Per stream: `open` → `active` → `closed-graceful` | `closed-abnormal`. No separate state machine — the adapter's state is implied by the libp2p stream's state.

## DataFrame (file transfer)

Delivered over the **data** stream protocol (`/neuron/browser-profile/data/1.0.0`). Identical to Spec 009's frame protocol.

| Frame | Payload | Source FR |
|-------|---------|-----------|
| Frame 0 (metadata) | Canonical JSON: `{ filename, sizeBytes, contentType, sha256Hex }` | FR-B19 |
| Frames 1..N (chunks) | Raw bytes of the file, up to 4 MiB per frame | FR-B19, FR-B20 |
| Zero-length | Keep-alive; silently consumed | — |

**Validation**:
- Metadata's `sizeBytes` MUST be ≤ 1 MiB (FR-B21 v1 cap). Larger → `NEURON-BROWSER-101`, abort before any chunk is read.
- Reassembled chunk-total MUST equal `sizeBytes`; mismatch → `NEURON-BROWSER-102`.
- Reassembled `sha256(payload)` MUST equal metadata `sha256Hex`; mismatch → `NEURON-BROWSER-082`.
- No activity for > `READ_IDLE_MS` (15 s Phase 1, ≤ 30 s per FR-B34) → `NEURON-BROWSER-121`.

## ConnectionSetupEnvelope (consumed by browser)

Spec 008 `connectionSetup` message extended per Spec 009. Browser consumes it; seller produces it.

| Field | Type | Description | Source FR |
|-------|------|-------------|----------|
| `recipientEVMAddress` | string (EIP-55) | Must equal the browser's own `evmAddress`. | FR-B15 |
| `encryptedMultiaddrs` | string (base64) | ECIES(secp256k1 ECDH + HKDF-SHA256 + AES-256-GCM, info `"neuron-multiaddr-v1"`) ciphertext of a JSON-encoded string-array of multiaddrs. | FR-B16 |
| `stream-protocol` | string | MUST equal the bootstrap's `dataStreamProtocolID`. | FR-B11 |

**Browser processing**:
1. Verify envelope signature + sender pinning (general rules).
2. ECIES-decrypt `encryptedMultiaddrs` using browser private key.
3. Parse decrypted value as `string[]` of multiaddrs.
4. MUST exercise the decrypt path even if the decrypted value equals the bootstrap multiaddr already in use (FR-B16).
5. Open the data stream on the first viable multiaddr; fall back to the bootstrap multiaddr if decrypted list is empty (v1 simplification).

## MockEscrowState (seller-side only)

Not visible to the browser. Defined here because the seller's `paymentDetails` envelope content depends on it.

```
 ┌──────────┐  release() ┌──────────┐
 │ proposed │ ─────────▶ │ released │
 └──────────┘            └──────────┘
      │
      │ refund()
      ▼
 ┌──────────┐
 │ refunded │
 └──────────┘
```

- `proposed`: seller has quoted; not yet funded on-chain (spike never funds).
- `released`: seller has delivered the file and observed `invoiceAck`.
- `refunded`: terminal fault state; not exercised in Phase 1's happy path.

Phase 1 exercises only `proposed → released`. Full lifecycle is Phase 2 (H1).

## Entity relationships (quick reference)

```
BootstrapJSON ────┐
                  │  read once, at session start
                  ▼
BrowserSession ───┬──▶ InStreamTopicAdapter ──▶ control libp2p stream
                  ├──▶ data libp2p stream ──▶ DataFrames (frame 0 + chunks)
                  ├──▶ BuyerState (runtime)
                  └──▶ LedgerEntry[] (append-only)

ConnectionSetupEnvelope  (inbound via InStreamTopicAdapter)
      │
      └─ decrypt ──▶ multiaddrs string[]  ──▶ host.dial ──▶ data stream
```
